## Issue: Milestone 3 - The LangGraph State Machine & Circuit Breakers

**Objective:**

Implement the core cognitive loop utilizing LangGraph. Construct the primary nodes, define strict conditional edges based on the Pydantic `GraphState`, and integrate the dual-layer anti-looping mechanisms to protect compute resources and prevent LLM tunnel vision.

**1. The Core Graph Workflow (`src/graph/workflow.py`)**

- Initialize the LangGraph utilizing the `GraphState` schema defined in Milestone 1.
- Create the three primary graph nodes:
    - `Node_Worker_Thinking`: The primary LLM reasoning engine that formulates plans and proposes tool calls.
    - `Node_Tool_Execution`: The execution routing layer (which will later interface with the Artifact Runner).
    - `Node_Review`: The QA and strategic oversight node.
- Define conditional edges: Route to `Node_Tool_Execution` if a tool is called, to `Node_Review` if the task is deemed complete, or suspend execution if user input is needed.

**2. Level 1 Protection: The Tool-Scanner (Micro-Loop)**

- Implement the deterministic hashing circuit breaker immediately before `Node_Tool_Execution`.
- **Logic:** When the Worker proposes a tool call, hash the tool name and exact parameters. Compare this hash against the `failed_tool_hashes` list in the state.
- **Action:** If a match is found, immediately block execution, increment `tool_error_count`, and return the system string: *"System block: This exact tool call already failed. Generate a completely new approach."* Do not consume LLM tokens for this check.

**3. Level 2 Protection: The Constitution Node (Macro-Loop)**

- Implement the Overseer logic within the `Node_Review`.
- **Logic:** If the `tool_error_count` hits the hard limit (e.g., 3 consecutive failures), route the state to the Overseer instead of the Intervention Queue.
- **Action:** The Overseer (a separate LLM call with a strict "Constitution" system prompt) analyzes the failure history. It does not solve the task but injects a mandatory strategic directive (e.g., `<SYSTEM_OVERRIDE>`) into the `message_history` to break the Worker's tunnel vision.

**4. Proactive Clarification (Anti-Hallucination)**

- Embed logic within `Node_Worker_Thinking` to evaluate if essential parameters are missing from the prompt.
- If data is missing, the node must halt execution, change the task status to `REQUIRES_CLARIFICATION`, and output a targeted question for the user (to be stored in the database for the UI to display).

**Acceptance Criteria:**

- The LangGraph compiles successfully and can execute a mocked workflow in the terminal without UI integration.
- **Test 1 (Hashing):** Providing a mocked repeated tool failure instantly triggers the Tool-Scanner block and appends the system warning to the message history.
- **Test 2 (Overseer):** Simulating 3 consecutive tool failures correctly routes the state to the Constitution Node, which successfully injects an override directive.
- **Test 3 (Clarification):** Submitting a task with deliberately missing parameters halts the graph and correctly updates the state to `REQUIRES_CLARIFICATION`.
