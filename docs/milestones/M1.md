## Issue: Milestone 1 - Core Environment, Pydantic Contracts & Unified Database

**Objective:**
Establish the foundational project structure, enforce strict data integrity through Pydantic models, and initialize the single-source-of-truth PostgreSQL database connection.

**1. Environment Initialization**

- Create a standard Python virtual environment (`python -m venv venv`).
- Define dependencies utilizing a clean `requirements.txt` (production) and `requirements-dev.txt` (local development/testing).
- Utilize standard `pip` for package management to maintain a streamlined and easily reproducible project structure.
- Install core dependencies: `pydantic`, `psycopg2-binary` (or `asyncpg`), `langgraph`, and `fastapi`.

**2. Pydantic Contracts (`src/core/models.py`)**

- Implement the Zero-Trust data layer by defining strict schema contracts.
- **`TaskObject`**: Must include status enums, specifically defining `REQUIRES_CLARIFICATION` for the proactive anti-hallucination protocol.
- **`GraphState`**: Must include the `tool_error_count` integer and the `failed_tool_hashes` list to support the Level 1 Anti-Looping mechanisms.
- **`QueueMessage`**: Must define the payload schema for routing tasks between the 7 designated PostgreSQL queues.

**3. Database Initialization (`src/memory/database.py`)**

- Implement the connection engine for the unified PostgreSQL database.
- Include the initialization logic to enable the `pgvector` extension natively upon connection.
- Draft the initial SQL table schemas representing the 7 system queues (User Input, Tasks, Error, Pending, Notification, Review, Result Output).

**Acceptance Criteria:**

- The environment builds cleanly via `pip install -r requirements-dev.txt` with zero dependency conflicts.
- A test script successfully instantiates the Pydantic models and intentionally fails when provided with invalid data types.
- The database module successfully connects to a local PostgreSQL instance and confirms `pgvector` is active.
