By building the FastAPI and WebSocket layers, we create the agnostic webhook ingress (allowing n8n, Tines, or simple CRON jobs to trigger the OS) and establish the real-time telemetry pipelines required for our `dwm`-style WebUI.

## Issue: Milestone 5 - API, Telemetry & Agnostic Ingress (FastAPI & WebSockets)

**Objective:**

Establish the communication layer of the Agentic OS using FastAPI. Enable external systems to trigger workflows via vendor-neutral webhooks, and implement WebSocket endpoints to stream real-time telemetry and state changes to the upcoming WebUI.

**1. Agnostic REST Ingress (`src/api/server.py`)**

- Initialize a FastAPI application.
- Create a primary webhook endpoint (e.g., `POST /api/v1/tasks/submit`).
- **Strict Validation:** The endpoint must accept a JSON payload and strictly validate it against the Pydantic `TaskObject` schema defined in Milestone 1.
- **Routing:** If validation passes, insert the task directly into the PostgreSQL `User Input Queue` and return an HTTP 202 Accepted with a generated Task ID.
- **Rejection:** If validation fails (e.g., missing fields, wrong data types), automatically reject with an HTTP 422 Unprocessable Entity. Do not allow malformed data to enter the database.

**2. Real-Time Telemetry (Semantic Terminal Feed) (`src/api/websockets.py`)**

- Implement a WebSocket endpoint (e.g., `ws://.../api/v1/stream/notifications`).
- **Logic:** This socket will tail the PostgreSQL `Notification Queue`. As the `Node_Worker_Thinking` evaluates steps or the Artifact Runner executes code, push these internal logs and thoughts over the socket in real-time.
- **Formatting:** Wrap the stream data in a strict JSON format so the future UI can easily parse and color-code different log levels (Info, Warning, Tool Execution, Error).

**3. Interactive UI Socket (Center Panel Connection)**

- Implement a bidirectional WebSocket endpoint (e.g., `ws://.../api/v1/stream/chat`) for the main user interface.
- **Logic:** This socket must handle synchronous user commands (like `/session`, `/clear`) and natural language prompts.
- **State Bridge:** When a task is parked in the `REQUIRES_CLARIFICATION` state (from Milestone 3), broadcast the clarification request through this socket, and listen for the user's reply to unpause the LangGraph execution.

**Acceptance Criteria:**

- **Test 1 (Ingress):** Sending a valid `curl` POST request to the API successfully writes a new task into the PostgreSQL database.
- **Test 2 (Validation):** Sending a malformed JSON payload instantly returns a 422 error without crashing the server or polluting the DB.
- **Test 3 (Telemetry):** Connecting a simple WebSocket test client (e.g., via Postman or wscat) successfully receives a continuous stream of mocked log events mimicking the LangGraph execution.
