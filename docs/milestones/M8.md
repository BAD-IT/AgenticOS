# Milestone 8: Production Hardening & Observability

## Objective
Harden the system for reliability, improve observability, and eliminate blocking patterns that caused task timeouts and poor UX.

## Deliverables

### 8.1 Container Reliability
- **PYTHONUNBUFFERED=1** on all containers — ensures logs appear immediately in `docker logs`
- **restart: unless-stopped** on worker, orchestrator, artifact_runner — auto-recovery from crashes
- **Health endpoint** (`GET /api/v1/health`) — Docker healthcheck probes DB connectivity, enables monitoring

### 8.2 Async Tool Execution with Per-Tool Timeout
- `node_tool_execution` is now **async** — uses `asyncio.to_thread()` + `asyncio.wait_for()`
- Each tool call has an independent **60-second timeout** (`TOOL_TIMEOUT_SECONDS`)
- Tool timeouts are isolated from the overall task timeout — a slow tool doesn't silently consume the full budget
- Eliminates the event loop blocking caused by `time.sleep()` in `run_in_sandbox`

### 8.3 Structured JSON Logging with Correlation IDs
- `JSONFormatter` outputs machine-parseable JSON to log files
- `current_task_id` context variable propagates task IDs across async boundaries
- Every log entry from worker, orchestrator, and sandbox includes `task_id`
- Human-readable stream handler for `docker logs` output
- Enables cross-container log correlation and automated log analysis

### 8.4 Intelligent Skill Generation Filter
- Experience consolidation only triggers for **meaningful tasks**:
  - Tasks that used tools (sandbox, file I/O)
  - Tasks with >3 messages (multi-step interaction)
- Simple Q&A ("hello", "what time is it") skip the LLM skill generation call
- Saves ~15 seconds of LLM time per trivial task

### 8.5 Modern Thinking Block UI
- Collapsible Claude/ChatGPT-style "Thinking..." block in the chat
- Real-time thinking phases: analyzing → tool execution → review → result
- Debug trace events injected as thinking steps with color-coded dots
- Token preview inside thinking block (suppresses raw XML)
- Smooth CSS animations: glow while active, collapse on completion
- Click to expand/collapse thinking details anytime

### 8.6 Sandbox Pipeline Fixes
- Runner daemon skips already-processed files (prevents infinite retry loop)
- Worker cleans up inbox task files after reading results
- Volume mount permissions corrected (worker inbox :rw, runner inbox :ro)
- Environment variables for absolute paths prevent WORKDIR resolution issues

### 8.7 Worker Merged into Orchestrator
- The standalone `worker` container is eliminated — **4 containers → 3 containers**
- `run_worker()` now launches as an `asyncio.create_task()` inside the FastAPI lifespan
- The worker loop shares the same event loop as the API and WebSocket handlers
- On shutdown, the worker task is cancelled gracefully
- Benefits: no duplicated Python runtime (~150MB saved), instant task pickup (no DB polling latency for initial dispatch), simpler debugging (one container, one log stream)
- The worker module (`src/core/worker.py`) remains unchanged — can be re-separated for horizontal scaling later

## Architecture Impact
- Reduced from 4 containers to 3 containers (db, orchestrator, artifact_runner)
- Backward-compatible with existing database schema
- All changes are additive — existing tools and workflows continue to work
