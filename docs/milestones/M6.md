## Issue: Milestone 6 - The Tiling WebUI (`dwm`Style) & Canvas Mode

**Objective:**

Develop a minimalist, keyboard-centric frontend architecture that prioritizes speed and visibility. Implement the 3-panel tiling layout, command-driven session management, WebSocket data streams, and the dynamic Canvas mode for artifact rendering.

**1. The 3-Panel Tiling Layout (`ui/index.html`, `ui/style.css`, `ui/app.js`)**

- Build a static, lightweight frontend (HTML/CSS/Vanilla JS) with zero heavy framework bloat.
- **Layout Structure:**
    - **Left Panel (Context & Canvas):** Displays active workspace variables, file trees, or toggles into an `iframe` for artifact rendering.
    - **Center Panel (Interface):** The primary CLI/Chat input area. Must capture keyboard focus by default.
    - **Right Panel (Semantic Terminal):** A scrolling, read-only terminal UI for real-time telemetry.
- **Keyboard Navigation:** Implement `dwm`style shortcuts (e.g., `Alt+1` through `Alt+0` to switch between 10 isolated Workspaces). Ensure complete navigability without a mouse.

**2. WebSocket Integration & State Handling**

- **Semantic Terminal (Right):** Connect to the Telemetry WebSocket established in Milestone 5. Render incoming JSON logs with appropriate color-coding (e.g., Green for success, Yellow for tool execution, Red for errors).
- **Command Interface (Center):** Connect to the Chat WebSocket. Handle standard chat interactions and implement the `REQUIRES_CLARIFICATION` state (pause the UI and clearly display the agent's follow-up question until the user replies).

**3. Native Quick-Step Commands**

- Intercept specific CLI inputs locally before they are sent to the heavy LLM engine to save compute:
    - `/clear`: Wipes the Center panel UI history.
    - `/session`: Displays current workspace session metadata.
    - `/stats`: Shows local hardware/VRAM usage.
    - `/joke`: Returns a random joke instantly via local static logic (per user preference).
    - `/time`: Returns the current local system time instantly (per user preference).

**4. Canvas Mode (Artifact Rendering)**

- Implement a toggle mechanism for the Left Panel.
- When the Orchestrator generates a web artifact (HTML/JS/React) and serves it via a local port on the `artifact_runner`, the UI must swap the Left Panel's context view for an `iframe` pointing to that port.
- Ensure the iframe is sandboxed to prevent XSS interactions with the main WebUI.

**Acceptance Criteria:**

- **Test 1 (UI/UX):** The UI loads in a strict 3-panel grid, and the user can switch between Workspaces 1-10 using only keyboard shortcuts. Workspace contexts do not bleed into each other.
- **Test 2 (Commands):** Typing `/time` and `/joke` in the Center panel returns immediate responses without triggering a LangGraph/LLM backend process.
- **Test 3 (Telemetry):** Triggering a backend task successfully streams live execution logs into the Right panel.
- **Test 4 (Canvas):** The Left panel successfully renders a simple local HTTP endpoint via iframe when toggled to Canvas Mode.
